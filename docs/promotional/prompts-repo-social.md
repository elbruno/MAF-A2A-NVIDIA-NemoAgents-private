# Repository Social Prompts — Detailed

Use `1200x630` for GitHub/OpenGraph/social cards.

## Prompt A (recommended, OpenGraph optimized)

```text
Create a 1200x630 OpenGraph-style social card for an open-source repository named "MAF-A2A-NVIDIA-NemoAgents".

MESSAGE TO COMMUNICATE:
- This is a practical multi-agent system demo.
- NeMo performs analysis.
- MAF performs actions.
- A2A links both through a web orchestration layer.

LAYOUT SPEC:
- Left third: analytical visuals (charts, anomaly spikes, KPI cards).
- Center third: orchestration/chat interface with routing arrows.
- Right third: action automation visuals (alerts, reports, execution states).
- Add subtle telemetry traces and service-health cues in the background.

VISUAL STYLE:
- Professional OSS project card, clean and technical.
- High contrast for small preview readability.
- Dark-blue enterprise palette with cyan highlights.
- Keep scene balanced and uncluttered.

DO NOT:
- Add watermarks, logos, fake paragraphs, or irrelevant symbols.
- Use cartoonish iconography or noisy collage style.
```

## Prompt B (minimal + bold)

```text
Generate a bold, minimalist 1200x630 social preview image for a GitHub repository about agent-to-agent communication.

DESIGN PRINCIPLES:
- Strong horizontal flow left-to-right: analysis -> orchestration -> action.
- Geometric UI-inspired modules, with elegant connector lines and small status indicators.
- Subtle reference to Aspire observability (dashboard-like cards and trace paths).
- Clear visual storytelling even when viewed as a small link preview.

MOOD:
- Engineering-focused, trustworthy, modern.
- Sleek enterprise visual system, no gimmicks.

NEGATIVE PROMPT:
- No photoreal humans, no mascots, no logos, no noisy text artifacts.
```
